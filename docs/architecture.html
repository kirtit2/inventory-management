Request / Data Flow
→
Read Flow (e.g. loading Dashboard)
1
User navigates to / — Vue Router mounts Dashboard.vue
2
onMounted reads current filter state from useFilters singleton composable
3
api.js fires GET /api/dashboard/summary?warehouse=&category=… via Axios
4
FastAPI receives request, runs filter helpers against in-memory JSON arrays
5
Returns filtered JSON; Vue renders stat cards and tables reactively
6
User changes a FilterBar dropdown → reactive state triggers a new API call, view re-renders
+
Write Flow (e.g. creating a Task)
1
User opens TasksModal via ProfileMenu and submits a new task
2
addTask() in App.vue calls api.createTask(data)
3
api.js sends POST /api/tasks with JSON body
4
FastAPI validates body via Pydantic, appends to in-memory list, returns new task object
5
New task is unshift-ed onto apiTasks ref — modal list updates instantly
6
Data is in-memory only — refreshing the server resets all created tasks

Key Design Patterns
🧩
Frontend Patterns
🔗
Singleton Composables — useFilters and useI18n hold module-level ref()s, so all components share the same reactive state without a store (Vuex/Pinia)
📊
Global FilterBar — one persistent FilterBar.vue in the shell drives query params for every view; views call getCurrentFilters() on mount and on filter change
🔀
API Layer — all Axios calls are centralised in api.js; views never import Axios directly, keeping HTTP concerns in one place
🌐
Hybrid Task List — App.vue merges mock tasks from useAuth with live API tasks via a computed ref; UI treats them identically
⚙️
Backend Patterns
📦
In-Memory Data Store — JSON files loaded once at import time into Python module-level lists; no ORM or database round-trips
🔍
Shared Filter Helpers — apply_filters() and filter_by_month() are reused across all endpoints, keeping filtering logic DRY
✅
Pydantic Models — each resource (InventoryItem, Order, BacklogItem…) has a typed Pydantic model; FastAPI auto-validates and auto-documents via /docs
🔢
Quarter Mapping — QUARTER_MAP dict translates Q1-2025…Q4-2025 filter values into month prefixes for flexible date filtering
